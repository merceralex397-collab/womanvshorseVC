### Scene Structure — Sound

- `wave_spawner.tscn`: Node3D root (WaveSpawner) + Timer child (SpawnTimer, one_shot=true, wait_time=2.0)
- Timer connected to `_on_spawn_timer_timeout()` via `@onready` — Godot 4.x idiomatic
- No spatial role on WaveSpawner Node3D; owns wave logic only — correct separation of concerns

### Script Design — Sound

- **State machine** (line 48): `IDLE → SPAWNING → WAVE_ACTIVE → WAVE_CLEARED → ALL_COMPLETE` — correctly specified, matches transition flow in plan
- **Signal emissions**: `wave_started` at spawn (line 111), `wave_cleared` when alive counter hits 0 (line 149), `all_waves_complete` on final wave done (line 97)
- **alive_enemies counter**: decremented in `_on_enemy_died` (line 146), `wave_cleared` fires only when `alive_enemies <= 0 AND _state == WAVE_ACTIVE` — state guard prevents double-fire
- **Enemy wiring**: `enemy.died.connect(_on_enemy_died)` per spawned instance; callable-based `.connect()` is Godot 4.x type-safe pattern
- **Navigation target**: `enemy.set_navigation_target(player_ref.global_position)` with `has_method` guard
- **Per-wave stat application**: `set_speed(speed)` and `set_max_health(health)` via `has_method` guards — correct, avoids hard assumption on HorseBase API

### Arena Edge Position Randomization — Correct

- 4 edges: east (+X), west (-X), north (+Z), south (-Z)
- `randi() % 4` selects edge; perpendicular axis uses `randf_range(-radius, radius)`
- Y=0.0 for arena floor — matches XZ-plane top-down design
- `spawn_radius` 9.0–11.0 stays within arena ±10 bounds from Camera3D size=10
- Plan explicitly notes `randf_range` on perpendicular axis and Y=0.0 — no ambiguity
