## 3. Prototype-Grade Visuals Identified

1. **Player invisible**: player.tscn has collision primitives (CapsuleShape3D, SphereShape3D) but no MeshInstance3D. The woman-warrior.glb model exists but is not wired into the player scene. Player appears as invisible capsule collision in-game.
2. **Missing models**: sword-projectile.glb (~200 tris) and heart-pickup.glb (~100 tris) do not exist in assets/models/. The spec requires all 8 models.
3. **No arena border**: arena-ground.glb is a flat plane with no fence or boundary visual. The arena looks unbounded and prototype-grade.
4. **main.tscn not in game flow**: main.tscn contains prototype BoxMesh/CapsuleMesh placeholders (Ground, WomanWarrior, HorseBrown) but is not loaded by the game — title_screen loads arena.tscn directly. Non-blocking but confirms placeholder artifacts exist outside game flow.

## 4. Implementation Steps

### Step 1: Add player visual model (HIGH PRIORITY — player is invisible)

- Add a `ModelContainer` → `PlayerMesh` Node3D/MeshInstance3D child to `player.tscn`
- Add `load_visual_model()` method to `player_controller.gd` using GLTFDocument to load `res://assets/models/woman-warrior.glb` into the PlayerMesh node, called from `_ready()` after stat initialization
- This mirrors the horse_base.gd pattern (Option A timing: variant name set before add_child, load called after)
- Verify: `godot --headless --path . --quit` exits 0 with player scene loaded

### Step 2: Generate missing GLB models via Blender-MCP

Two models missing from the 8-model requirement:

**heart-pickup.glb** (~100 tris):
- Blender-MCP chain: project_initialize(output_blend=/tmp/heart-pickup.blend) → scene_batch_edit (scaled ICO_SPHERE or merged primitives forming heart silhouette) → material_pbr_build(Red #E53935, slight emissive) → quality_validate → export to res://assets/models/heart-pickup.glb
- AC: model exists, ≤100 tris, manifold, imports cleanly, PROVENANCE.md updated

**sword-projectile.glb** (~200 tris):
- Blender-MCP chain: project_initialize(output_blend=/tmp/sword-projectile.blend) → scene_batch_edit (elongated blade: rotated CYLINDER + handle BOX) → material_pbr_build(Silver #C0C0C0 blade, Brown #795548 handle) → quality_validate → export to res://assets/models/sword-projectile.glb
- AC: model exists, ≤200 tris, manifold, imports cleanly, PROVENANCE.md updated

### Step 3: Arena border / fence dressing

- Use Godot BoxMesh3D instances (no new Blender-MCP work needed for fence — fast scene work)
- Create fence posts as thin vertical BoxMesh3D placed around arena perimeter
- Create rails as thin horizontal BoxMesh3D connecting posts
- Material: Dark wood color (#5D4037) via StandardMaterial3D
- AC: arena has visible boundary in game view — player can see the fence edge
