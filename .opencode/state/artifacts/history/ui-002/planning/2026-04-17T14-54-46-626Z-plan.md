# Planning Artifact — UI-002: Game Over Screen

## 1. Scope

Create `game_over.tscn` and `game_over.gd` implementing a game-over / victory screen with:
- Full-screen Control root with CenterContainer layout
- "GAME OVER" or "VICTORY" label controlled by a `victory: bool` property
- Final score and wave-reached display
- "Play Again" and "Main Menu" touch-friendly buttons
- Scene transitions to arena (Play Again) and title screen (Main Menu)

## 2. Files / Systems Affected

| File | Role |
|------|------|
| `scenes/ui/game_over.tscn` | Game-over/victory scene (new) |
| `scripts/ui/game_over.gd` | Screen logic, victory flag, button signals (new) |
| `scenes/ui/title_screen.tscn` | Structural reference (UI-001, done) |
| `project.godot` | No changes required |

## 3. Scene Structure Design

```
game_over.tscn
└── Control (root, full-screen, anchors to margins)
    └── CenterContainer
        └── VBoxContainer (alignment: CENTER, separation: 16px)
            ├── Label: result_text ("GAME OVER" / "VICTORY")
            │         Custom fonts & large size for visual impact
            ├── Label: "SCORE: {score}" (score_display)
            ├── Label: "WAVE: {wave}" (wave_display)
            ├── MarginContainer (vertical spacer, 24px)
            ├── Button: "Play Again" (play_again_btn, min size 200×64)
            └── Button: "Main Menu" (main_menu_btn, min size 200×64)
```

**Layout rationale:**
- `Control` root anchors to all four edges for true full-screen coverage
- `CenterContainer` centers content both axes
- `VBoxContainer` stacks elements vertically with consistent spacing
- Buttons have `min_height = 64px` for touch-friendly targets (64px > 48dp minimum)
- Two spacer `Control` nodes (`MarginContainer` or plain `Control` with custom height) pad between stats and buttons

## 4. Script Design

### `scripts/ui/game_over.gd`

```gdscript
extends Control

@export var victory: bool = false  # false = Game Over, true = Victory

@onready var result_label: Label = $CenterContainer/VBoxContainer/result_text
@onready var score_label: Label = $CenterContainer/VBoxContainer/score_display
@onready var wave_label: Label = $CenterContainer/VBoxContainer/wave_display
@onready var play_again_btn: Button = $CenterContainer/VBoxContainer/play_again_btn
@onready var main_menu_btn: Button = $CenterContainer/VBoxContainer/main_menu_btn

func _ready() -> void:
    # Set result text based on victory flag
    if victory:
        result_label.text = "VICTORY"
    else:
        result_label.text = "GAME OVER"
    
    # Populate stats — read from GameState singleton if available,
    # otherwise display 0 as fallback
    var final_score: int = 0
    var final_wave: int = 1
    if Engine.has_singleton("GameState"):
        var gs = Engine.get_singleton("GameState")
        final_score = gs.get("score", 0)
        final_wave = gs.get("wave", 1)
    
    score_label.text = "SCORE: %d" % final_score
    wave_label.text = "WAVE: %d" % final_wave
    
    # Wire button signals
    play_again_btn.pressed.connect(_on_play_again)
    main_menu_btn.pressed.connect(_on_main_menu)

func _on_play_again() -> void:
    get_tree().change_scene_to_file("res://scenes/arena/arena.tscn")

func _on_main_menu() -> void:
    get_tree().change_scene_to_file("res://scenes/ui/title_screen.tscn")
```

**Design decisions:**
- `victory` is `@export` so the calling scene (e.g. `arena.tscn` or a game-state script) can set it before or after instancing
- Stats read from `GameState` singleton with `0`/`1` fallback so the screen is always populated even if called without game-state wiring
- Both buttons use Godot 4 `get_tree().change_scene_to_file()` for scene transitions
