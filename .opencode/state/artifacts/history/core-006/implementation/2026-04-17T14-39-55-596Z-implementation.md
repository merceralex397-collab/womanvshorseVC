### Step 4 — Added `load_variant_model()` public method to `scripts/enemies/horse_base.gd`

```gdscript
func load_variant_model() -> void:
    var variant_data: Dictionary = HorseVariants.get_variant(variant_name)
    if variant_data.is_empty():
        push_warning("HorseBase: unknown variant_name '" + variant_name + "', no model loaded")
        return
    var model_path: String = variant_data.get("model_path", "")
    # GLTFDocument.append_from_file() loads the GLB into the MeshInstance3D child
    var state := GLTFState.new()
    var doc := GLTFDocument.new()
    var result := doc.append_from_file(model_path, state)
    if result != OK:
        push_error("HorseBase: GLTFDocument.append_from_file failed for " + model_path)
        return
    if state.get_mesh_count() > 0:
        mesh_instance.mesh = state.get_mesh(0)
```

## Option A Timing (key ordering)

```
1. enemy = enemy_scene.instantiate()
2. enemy.variant_name = variant   ← BEFORE add_child
3. _enemy_container.add_child(enemy)
4. enemy.speed = speed
5. enemy.max_health = health
6. enemy.set_navigation_target(...)
7. enemy.load_variant_model()      ← AFTER add_child; variant_name already set
```

This ensures `load_variant_model()` uses the correct `variant_name` when looking up the GLB path, avoiding the timing defect (all enemies loading brown GLB regardless of variant) identified in the earlier plan review.
