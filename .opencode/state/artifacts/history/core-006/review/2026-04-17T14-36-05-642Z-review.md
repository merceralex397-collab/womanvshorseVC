## 2. Timing Fix Verification ✓

### Prior blocking defect (REV-1) — RESOLVED ✓

**Root cause (prior review)**: `_load_variant_model()` was called inside `_ready()`, but `wave_spawner.gd` set `variant_name` **after** `add_child`. Every enemy loaded the brown GLB regardless of variant assignment.

**Option A fix in updated plan**:
- `load_variant_model()` is now a **public method** (not `_load_variant_model` with underscore) — no auto-call in `_ready()`
- wave_spawner's `_spawn_enemy` assigns `enemy.variant_name = variant` **before** calling `enemy.load_variant_model()`
- Explicit corrected sequence in plan §4.3:
  ```
  _enemy_container.add_child(enemy)   ← _ready() fires (no GLB loading)
  enemy.variant_name = variant       ← set FIRST
  enemy.speed = speed
  enemy.max_health = health
  enemy.load_variant_model()         ← called AFTER all properties set → CORRECT variant
  ```

**Verdict**: Option A correctly eliminates the timing ambiguity. `_ready()` no longer triggers GLB loading; the explicit call chain guarantees `variant_name` is set before `load_variant_model()` reads it. No hidden frame-order dependency.

### Option A vs Options B/C — rationale sound ✓

Plan §4 correctly reasons:
- Option B (`call_deferred`): still requires `variant_name` to be set before the deferred call; adds a frame delay and doesn't solve the ordering problem
- Option C (`defer_model_load` flag): adds state flag to keep in sync; more complex than needed
- Option A is explicit: caller controls exactly when GLB loading occurs after property setup

### GLTFDocument API ✓

`GLTFDocument.append_from_file` + `generate_state().meshes[0].mesh` → `MeshInstance3D` is the correct Godot 4.6 runtime GLB loading pattern. Plan §5.2 step-by-step code is sound.

### Skeleton root handling ✓

Plan extracts `state.meshes[0].mesh` and assigns to a fresh `MeshInstance3D` named `"VariantMesh"` as a child of the horse_base Node3D. This bypasses the GLB skeleton root correctly. CollisionShape3D remains the physics body; GLB is visual-only. Acceptable for prototype.
