## 1. Timing Fix Verification — BLOCKER-1: RESOLVED ✅

**Evidence from plan text (§4 and §5.2)**:
- `load_variant_model()` is a **public method** (no underscore prefix) defined in `horse_base.gd`
- It is **NOT called from `_ready()`** — the plan explicitly states `_ready()` fires without GLB loading (line 131: "NOTE: GLB loading is NOT called here")
- The corrected spawn sequence in `_spawn_enemy` (lines 176-189) sets `variant_name` **before** calling `load_variant_model()`:
  ```
  enemy.variant_name = variant    ← set FIRST
  enemy.speed = speed
  enemy.max_health = health
  enemy.load_variant_model()      ← called AFTER variant_name is set
  ```

**Current codebase evidence**:
- `scripts/enemies/horse_base.gd` `_ready()` (lines 19-24) contains **no GLB loading** — confirms the plan's removal is accurate for the current state
- `scripts/wave_spawner/wave_spawner.gd` `_spawn_enemy` (lines 97-131) currently has no variant handling — confirms plan modifications are still needed (expected at plan stage)

**Verdict**: Timing defect is **resolved**. No hidden timing dependencies remain. The explicit call chain is the correct pattern.
