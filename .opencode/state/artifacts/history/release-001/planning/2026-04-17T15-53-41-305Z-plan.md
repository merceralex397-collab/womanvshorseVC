## 2. Path Discrepancy Analysis

| Source | APK path | Case |
|---|---|---|
| ANDROID-001 (done/trusted) | `build/android/womanvshorseVC-debug.apk` | capital VC |
| RELEASE-001 AC-2 | `build/android/womanvshorsevc-debug.apk` | lowercase vc |

**Root cause**: ANDROID-001's `export_presets.cfg` was configured with `womanvshorseVC-debug.apk` (capital VC). RELEASE-001's acceptance criteria were written with `womanvshorsevc-debug.apk` (all lowercase) — likely a case mismatch introduced when the ticket was split from ANDROID-001.

**Resolution approach**: The plan resolves this discrepancy by re-running the Godot export targeting the lowercase path, which either produces a fresh APK at the correct path or demonstrates that a simple file copy/rename is sufficient to satisfy the AC. The re-export approach is preferred because it also satisfies AC-1 (the command succeeds).

## 4. Implementation Steps

### Step 1 — Check existing APK at both paths
- **Action**: Run `ls -la build/android/womanvshorsevc-debug.apk 2>/dev/null && echo EXISTS || echo MISSING` (lowercase)
- **Action**: Run `ls -la build/android/womanvshorseVC-debug.apk 2>/dev/null && echo EXISTS || echo MISSING` (capital VC, from ANDROID-001)
- **Pass condition**: At least one exists; report both sizes

### Step 2 — Probe Godot binary availability
- **Action**: Run `which godot4 && godot4 --version` and `which godot && godot --version` to find the resolved binary
- **Pass condition**: At least one binary responds; record the exact path used

### Step 3 — Run Godot Android debug export to lowercase path
- **Action**: Run:
  ```
  godot4 --headless --path . --export-debug "Android Debug" build/android/womanvshorsevc-debug.apk
  ```
  (or the exact binary path resolved in Step 2, with the same `--path .` argument from AC-1)
- **Pass condition**: Exit code 0; APK produced at `build/android/womanvshorsevc-debug.apk`

### Step 4 — Verify APK file size
- **Action**: Run `ls -la build/android/womanvshorsevc-debug.apk`
- **Pass condition**: File exists, non-zero size

### Step 5 — Verify APK internal structure via unzip
- **Action**: Run `unzip -l build/android/womanvshorsevc-debug.apk`
- **Pass condition**: Output contains `AndroidManifest.xml` and either `classes.dex` or `classes[0-9]*.dex` or `classes/` directory

### Step 6 — Record the exact command that succeeded
- **Action**: Record in implementation artifact:
  - Exact binary path resolved (e.g. `/usr/bin/godot4`)
  - Exact arguments used (`--headless --path . --export-debug "Android Debug" build/android/womanvshorsevc-debug.apk`)
  - Exit code
  - APK size
