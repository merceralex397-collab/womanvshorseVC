## 3. Implementation Steps

### Step 1: Run Direct Command Verification During Review Stage

Run the following commands **during the review stage** and capture raw output:

**Command Set A — Godot Binary Verification:**
```bash
which godot
godot --version
```

**Command Set B — Godot Headless Validation:**
```bash
godot --headless --path /home/pc/projects/womanvshorseVC --quit
```

Capture stdout, stderr, and exit code for each command **directly in the review artifact**.

### Step 2: Generate Regenerated Review Artifact

Create a new review artifact at:
`.opencode/state/artifacts/history/remed-001/review/<timestamp>-review.md`

The regenerated review artifact must contain:

1. **Exact commands run** (verbatim, as typed):
   ```
   which godot
   godot --version
   godot --headless --path /home/pc/projects/womanvshorseVC --quit
   ```

2. **Raw stdout/stderr output** for each command (unmodified, direct capture):
   ```
   STDOUT: "/home/pc/.local/bin/godot"
   STDERR: ""
   EXIT_CODE: 0
   ```
   ```
   STDOUT: "4.6.2.stable.official.71f334935"
   STDERR: ""
   EXIT_CODE: 0
   ```
   ```
   STDOUT: "Error: Can't run project: no main scene defined in the project.\n"
   STDERR: ""
   EXIT_CODE: 1
   ```

3. **Explicit PASS/FAIL result** stated in review body:
   - For each acceptance criterion: `**Result: PASS**` or `**Result: FAIL**`
   - Final verdict section must state: `**Verdict: APPROVE**` or `**Verdict: REJECT**`

4. **No citations to implementation artifact as primary evidence** — the review artifact must be self-contained with direct command output.

### Step 3: Register Regenerated Review Artifact

Use `artifact_register` to register the new review artifact with:
- `kind`: `review`
- `stage`: `review`
- `path`: the new timestamped path
- `ticket_id`: `REMED-001`
- `summary`: Note that the review now contains direct command evidence per EXEC-REMED-001 process requirements

### Step 4: Mark REMED-001 as Trustworthy for Reverification

After the regenerated review artifact is registered and passes review, run `ticket_reverify` on REMED-001 to restore its `verification_state` to `trusted` using the new review evidence.

### Step 5: Update Artifact Supersession
