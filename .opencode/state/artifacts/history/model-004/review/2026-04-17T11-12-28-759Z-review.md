## Root Cause Analysis

### The bridge is NOT defective

The Blender MCP bridge (`bridge_runtime.py`) is the integration layer between the MCP tool interface and Blender's command line. The audit log and command lines prove:

1. When `input_blend` is a non-null path string, the bridge places it **before** `--python` in the Blender command line, exactly where Blender expects a file path for `--background`
2. When `output_blend` is a non-null path string, the bridge writes to that path and returns its hash in the audit log
3. The same bridge handled MODEL-003's successful 14-call chain at 09:05–09:07

### The defect is caller-side

The implementer for MODEL-004 did not pass `input_blend` and `output_blend` parameters to the `scene_batch_edit` calls. The bridge received `null` for both, had nothing to forward, and executed Blender without a source blend file — resulting in a fresh empty scene each call.

### This is the same pattern as MODEL-003's initial failures

MODEL-003's first attempts (lines 21–40 in audit_20260417.jsonl, ~03:27–03:35) also show `scene_batch_edit` calls with `input_blend: null, output_blend: null`. Those also produced empty scenes and null persisted blend paths. The MODEL-003 implementer later corrected the chaining pattern and achieved the successful 09:05–09:07 chain.

---

## Correct Chaining Pattern (proven by MODEL-003)

The correct pattern is:

1. `project_initialize(output_blend=stage1.blend)` — creates a base scene
2. `scene_batch_edit(input_blend=stage1.blend, output_blend=stage2.blend, operations=[...])` — edits persist to stage2
3. Use `stage2.blend` as the next call's `input_blend`, chain to `stage3.blend`, and so on
4. Each call's `input_blend` must equal the **previous call's `output_blend`**

The implementer must:
1. Start with `project_initialize(output_blend=horse-war-stage1.blend)`
2. For each subsequent `scene_batch_edit` call, pass:
   - `input_blend` = the file returned by the previous step's `output_blend`
   - `output_blend` = a new stage file path
3. Use the final stage blend for `mesh_edit_batch`, `material_pbr_build`, `quality_validate`, and `export_asset`

## Findings

### Finding 1: CRITICAL — Caller did not pass blend chaining parameters

**Classification:** Caller-side error

**Evidence:**
- Audit line 220: `scene_batch_edit` for MODEL-004 has `input_blend: null, output_blend: null`
- Audit line 103: `scene_batch_edit` for MODEL-003 (same bridge, same session) has valid blend paths
- Command line comparison confirms MODEL-004's Blender invocation lacks the blend file path argument

**Resolution:** Return to implementation with the explicit blend chaining pattern. The implementer must pass the output of each step as the input to the next step, as proven by MODEL-003's successful 09:05–09:07 chain.

**Evidence:**

**Resolution:** Same as Finding 1. Once valid geometry is built with correct blend chaining, re-export and re-validate.
