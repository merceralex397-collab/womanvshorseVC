## Known Issues from MODEL-003 (Context)

MODEL-003 (horse-black) is blocked because the Blender MCP stateless bridge could not chain `scene_batch_edit` operations — specifically `create_primitive` auto-named everything as `Cube.XXX` ignoring the `object` parameter, and `set_transform` failed to find user-specified names. The implementation never progressed past probe stage.

**MODEL-004 plan sidesteps this by:**
1. Not attempting to reference user-specified object names in operations — only using auto-generated names (Cube, Cylinder, etc.) as they are observed
2. Structuring each `scene_batch_edit` call to do multiple related operations in a single call rather than depending on prior calls to create named objects
3. Using the same `input_blend` = `output_blend` pattern that worked for MODEL-002
