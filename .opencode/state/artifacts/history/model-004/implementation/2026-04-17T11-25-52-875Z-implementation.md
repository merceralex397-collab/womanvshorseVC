## Root Cause Analysis

| Issue | Evidence | Analysis |
|---|---|---|
| project_initialize works correctly | output_blend correctly forwarded | Tool framework correctly passes REQUIRED parameters |
| scene_batch_edit receives null | All calls show null in audit | Tool framework does NOT forward OPTIONAL parameters with default null |
| Explicit null vs omitted field | Both produce null | Bridge receives null either way |
| Bridge treats null as "no blending" | Warning messages confirm | Explicit null = "no persistence" to bridge |

The tool schema for `scene_batch_edit` defines `input_blend` and `output_blend` as:
```json
"input_blend": {"anyOf": [{"type": "string"}, {"type": "null"}], "default": null}
```

When `null` is passed (explicitly or by default), the bridge interprets this as "no blending needed" rather than "use factory startup without persistence". This is inconsistent with `project_initialize` where the required `output_blend` is correctly forwarded.
