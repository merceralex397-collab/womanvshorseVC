## Root Cause

Identical to MODEL-007 blocker documented in implementation artifact lines 43–55:

- `_wrap_v1_scene_batch_edit` in `bridge_runtime.py` does not forward `input_blend`/`output_blend` to the Blender `--background` command line
- Each `scene_batch_edit` call starts from `--factory-startup` (fresh state), discarding the `project_initialize` blend
- Audit log record 21: `input_blend: null, output_blend: null` — bridge dropped blend path forwarding
- Audit log record 23: `input_blend: null, output_blend: null` — same defect repeated
- This is a server-level regression, not an implementation error

**Audit evidence from `.blender-mcp/audit/audit_20260418.jsonl`:**
- Record 18 (project_initialize): `output_blend: "/home/rowan/womanvshorseVC/tmp/heart-pickup-stage.blend"` — blend written correctly ✅
- Record 21 (scene_batch_edit #1): `input_blend: null, output_blend: null` — blend paths dropped ❌
- Record 23 (scene_batch_edit #2): `input_blend: null, output_blend: null` — same defect repeated ❌
