## Root Cause Analysis

**Same root cause as MODEL-003 and MODEL-004 initial failures.**

The Blender MCP bridge is not forwarding `input_blend` and `output_blend` parameters to the Blender subprocess command line. Evidence:

- Audit log `job_start` records show `input_blend: null` and `output_blend: null` on ALL `scene_batch_edit` calls after `project_initialize`
- The bridge correctly persisted `init.blend` from `project_initialize` (proving write paths work)
- But subsequent calls do not receive the blend paths as Blender arguments

**This is a bridge-level defect, not a caller error.** The caller IS providing explicit paths. The bridge IS NOT forwarding them.
