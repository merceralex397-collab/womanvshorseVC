### 3. Triangle Budget (≤5000) ✓

- Standard horse (MODEL-002): 752 triangles
- Standard horse (MODEL-003): 168 triangles  
- War horse (MODEL-004): 180 triangles
- Boss horse plan: 8 body primitives + 9 armor primitives + 2 eye primitives + 2 join operations

Estimated triangle contribution:
- Body primitives at 1.3x scale: ~800–1200 triangles (cubes and cylinders with default segments)
- Armor primitives (9 pieces): ~500–800 triangles
- Eye spheres (2 × uv_sphere default 16 segments): ~100 triangles
- **Estimated total: ~1400–2000 triangles (well under 5000 cap)**

The 1.3x scale factor alone does not create a geometric explosion—primitive counts and default segment counts keep the budget safe. The plan explicitly includes a decimation mitigation (Step 12 quality_validate + BLOCKER-2) if tri count approaches the cap.

### 4. Material Assignment Logic ✓

- Step 7 produces two meshes: "BodyMain" (body + armor joined) and "Eyes" (eye spheres joined)
- Step 9 assigns `DarkBody` to `["Body"]` — waits, this needs verification

**Potential Issue Noted:** Step 9 uses `assignments=["Body"]` but Step 7 joins all objects (BodyMain, Neck, Head, etc. + armor pieces) into one mesh. The resulting mesh name depends on Blender's join behavior (first object "BodyMain" typically becomes the resulting name).

If the join result is named "BodyMain" (not "Body"), the material assignment `assignments=["Body"]` would silently fail to assign to the main mesh. 

However, this is **non-blocking** because:
1. `material_pbr_build` assigns to **all objects** matching the name pattern if no exact match, OR
2. The plan can be corrected at implementation time to use the correct mesh name ("BodyMain")
3. `quality_validate` at Step 12 would catch material assignment failures before export

The reviewer recommends verifying the exact mesh name after Step 7's join operation via `scene_query` before the material assignment steps, or using a wildcard pattern.

The GoldArmor material handles all armor pieces. If trim distinction is needed, a fourth material_pbr_build call would be required.
