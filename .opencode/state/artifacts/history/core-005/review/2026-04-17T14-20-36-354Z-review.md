### Friendly Fire Guard
Explicitly addressed in AC-6 and in the implementation of `_on_contact_body_entered`:
```gdscript
if body is CharacterBody3D and body.has_method("take_damage"):
    body.take_damage(contact_damage)
```
Enemies' `take_damage` lives on the `Health` Node child, not on the `CharacterBody3D` itself, so the guard prevents enemy-to-enemy damage through contact.

### Advisory Notes (Non-Blocking)

1. **BaseMaterial3D allocation in flash()**: `PlayerHitFlash.gd` and `HitFlash.gd` create a new `BaseMaterial3D` on every flash call. For frequent hits this has minor GC pressure. Non-blocking for the prototype stage; noted for VISUAL-001 review.

2. **Invincibility timing**: `_start_invincibility()` uses `await get_tree().create_timer(...).timeout`. If `take_damage` is called again before the coroutine completes, a second `await` begins, and the original timer is abandoned. The 0.5s window is sufficient for gameplay feel. Not a plan defect.

3. **Health.died wiring gap**: The current `horse_base.gd` implementation (from CORE-002) does not yet wire `Health.died` in `_ready()`. The plan correctly specifies this wiring as Step 7 and it must be implemented.
